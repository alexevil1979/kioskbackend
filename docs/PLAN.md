## Этапы и оценка времени

| # | Этап | Содержание | Часы (оценка) |
|---|------|------------|---------------|
| **0** | Уточнения и контракт API | Swagger/примеры ответов CRM, СБП, касса | 4–8 (ожидание заказчика) |
| **1** | Каркас проекта | Структура, config, logging, `requirements.txt` | 4 |
| **2** | Kiosk shell | Fullscreen, блокировка клавиш (Win API), автозапуск | 6–8 |
| **3** | State machine + навигация | `QStateMachine`, все экраны-заглушки | 6 |
| **4** | Дизайн-система | QSS, шрифты, цвета, компоненты (кнопки, карточки) | 8 |
| **5** | Главное меню | Категории, сетка товаров, +/- на карточке, нижняя панель | 12 |
| **6** | Корзина | Список, редактирование, итог | 8 |
| **7** | CRM + кэш | `CRMClient`, mock, polling 15–30 с, `media/products/` | 10 |
| **8** | Оплата СБП | QR, таймер, статус, отмена | 8 |
| **9** | Оплата картой | Интеграция Т-Банк (CLI/SDK) | 12–20 |
| **10** | Успех / ошибки / офлайн | Экраны 5–7, автовозврат 5–8 с | 4 |
| **11** | Бездействие | 50 с предупреждение, 60 с сброс | 4 |
| **12** | Стартовый экран | Splash / «Коснитесь экрана» | 2 |
| **13** | Печать чека | УМКА COM/HTTP | 16–24 |
| **14** | Тесты на железе | NUC, сенсор, принтер, POS, сеть | 16 |
| **15** | Деплой | Installer, автозапуск, логи, инструкция | 8 |

**Итого (разработка без ожидания API):** ~110–130 ч (~3–4 недели при 1 разработчике full-time).

**MVP (этапы 1–7 + заглушки оплаты):** ~58 ч (~1,5 недели).

## Что нужно запросить у заказчика (блокеры интеграций)

### CRM / каталог
- [ ] Base URL API (prod/test)
- [ ] Авторизация (API key, Bearer, basic)
- [ ] Эндпоинты: категории, товары, цены, остатки, фото (URL или binary)
- [ ] Формат JSON (пример ответа)
- [ ] ID точки продаж / киоска для заказа

### Оплата СБП
- [ ] Провайдер (Т-Банк, Сбер, ЮKassa, свой банк)
- [ ] API создания платежа + webhook/polling статуса
- [ ] Срок жизни QR и таймаут ожидания в ТЗ (секунды)

### POS Т-Банк
- [ ] Модель терминала и способ вызова (EXE, TCP, USB, Cloud API)
- [ ] Документация SDK / путь к `TinkoffAcquiringTerminal.exe` или аналог
- [ ] Тестовые реквизиты / sandbox

### Касса УМКА-01-ФА
- [ ] Протокол: COM (порт, baud) или HTTP (IP:port)
- [ ] Драйвер / библиотека / пример чека
- [ ] Система налогообложения, НДС, наименование организации

### Принтер HS-K33
- [ ] Печать через кассу или отдельно ESC/POS по сети?
- [ ] IP принтера в локальной сети

### Бизнес-правила
- [ ] Список категорий и товаров для MVP (или только API)
- [ ] Поведение при `stock = 0` (скрыть / «нет в наличии»)
- [ ] Нужен ли стартовый экран с логотипом (файл PNG/SVG)

### Kiosk / Windows
- [ ] Учётная запись Windows только под киоск (рекомендуется)
- [ ] Пароль администратора для «служебного выхода» (скрытая комбинация?)
- [ ] Разрешение экрана подтверждено: 1920×1080?

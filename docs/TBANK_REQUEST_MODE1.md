# Запрос в Т-Банк (Т-Бизнес): киоск самообслуживания, режим 1 — aQsi 6

**Версия:** 1.0  
**Дата:** 2026-05-29  
**Проект:** киоск на ферме, самообслуживание без кассира  
**Статус заявки:** подана  
**Режим интеграции:** `tbank_aqsi` (основной)  

Документ для передачи менеджеру Т-Банка / в чат Т-Бизнеса (**8 800 700-66-66**) или приложением к заявке на эквайринг.

---

## 1. Суть проекта

| Параметр | Значение |
|----------|----------|
| Тип точки | Стационарный киоск самообслуживания (ферма, розница) |
| Интерфейс | Сенсорный экран 32″, ПО на **Windows 10** (NUC) |
| Кассир | **Нет** (unattended) |
| Оплата | **aQsi 6** («3 в 1»: эквайринг + онлайн-касса + принтер чека) |
| Отдельная УМКА | **Не используется** |
| Отдельный киоск-принтер | **Не используется** (печать на aQsi) |
| Каталог товаров | Внешняя CRM по API (интернет) |

### 3.2. Интеграция киоск (NUC) ↔ aQsi

Киоск **не** использует кассу вручную. Нужен сценарий **без кассира**:

| № | Требование | Зачем |
|---|------------|--------|
| 1 | **API aQsi** (внешняя интеграция) — ключ и документация | Киоск создаёт заказ с позициями и суммой |
| 2 | Режим **автоматической / отложенной оплаты** без выбора кассиром в меню | Самообслуживание |
| 3 | Подтверждение от **api@aqsi.ru** или Т-Банка, что схема допустима для aQsi 6 + Т-эквайринг | Юридически и технически |
| 4 | Callback / polling **статуса оплаты** (оплачен / отменён / таймаут) | Экран «Спасибо» / ошибка на киоске |
| 5 | После оплаты — **фискальный чек** на aQsi (печать + ОФД) | 54-ФЗ |

## 5. Резервный режим (информационно)

На объекте возможен **запасной** режим `tbank_pos_printer` (POS PAX + принтер HS-K33 + облачная касса) **без УМКА**.  
Для текущего этапа приоритет — **только режим 1 (aQsi 6)**. Резерв согласуем отдельно при необходимости.

# Архитектура «Т-Банк first» (УМКА и принтер не обязательны)

Базовый принцип: **фискализация и бумажный чек — зона ответственности решения Т-Банка**, а киоск (NUC) — UI, каталог, корзина и инициация оплаты.

## Варианты от Т-Банка

| Вариант | Устройство | Оплата | Чек 54-ФЗ | Интеграция с киоском |
|---------|------------|--------|-----------|----------------------|
| **A (рекомендуется для киоска)** | [aQsi 6 — «3 в 1»](https://www.tbank.ru/business/payments/acquiring/terminal/) | Карта, QR на терминале, СБП | Встроенная ККТ + принтер | [aQsi Cloud API](https://api.aqsi.ru/) — отложенный заказ |
| **B** | PAX + **HS-K33**, **без УМКА** | Карта (Smart Sale), опц. СБП на киоске | **Облако** CloudKassir / [Чеки Т-Бизнеса](https://www.tbank.ru/business/online-payments/cashreg/) | [`08-tbank-pos-printer.md`](08-tbank-pos-printer.md) |
| **C** | Только экран киоска | **СБП** (QR на 32″) | Облачная касса после оплаты | [Т‑Касса eacq API](https://developer.tbank.ru/eacq/api) |

Источники: [терминалы Т-Банка](https://www.tbank.ru/business/payments/acquiring/terminal/), [онлайн-касса и 54-ФЗ](https://www.tbank.ru/business/help/business-payments/acquiring/online/business-needs/), [интеграция касса↔терминал](https://www.tbank.ru/business/help/business-payments/acquiring/online/integrate/).

### Что НЕ нужно при варианте A

- УМКА-01-ФА на switch
- HS-K33 (в aQsi 6 уже есть термопринтер чеков)

### Когда УМКА/принтер всё же имеют смысл

- Уже куплены и зарегистрированы
- Выбран **PAX без ККТ** (вариант B) и своя касса — не aQsi
- Нужен второй физический принтер у киоска (редко)

## Рекомендуемая схема для вашего объекта

```
                    ┌──────────────┐
                    │   Роутер    │
                    │  (интернет) │
                    └──────┬───────┘
           Wi‑Fi            │            Wi‑Fi / LAN
      ┌──────────┐          │       ┌─────────────┐
      │ NUC      │          │       │ aQsi 6      │
      │ киоск UI │◄─ HTTPS ─┼──────►│ (Т-Банк)    │
      │ PyQt6    │  aQsi API │       │ оплата+чек │
      └──────────┘          │       └─────────────┘
           │                │
           └── CRM, СБП* ────┘
               *СБП на экране киоска — опционально, если не используете QR на aQsi
```

**Поток (вариант A):**

1. Покупатель набирает корзину на NUC.
2. Киоск создаёт **заказ в aQsi API** (позиции, сумма, `externalId`).
3. На aQsi 6 появляется отложенный заказ (или автоматическая оплата — уточнить у aQsi/Т-Банка для unattended).
4. Покупатель платит на терминале (карта / QR / СБП на устройстве).
5. aQsi пробивает фискальный чек и печатает / отправляет e-mail/SMS.
6. Киоск по API опрашивает статус заказа → экран «Спасибо».

## Конфигурация в проекте

`config/settings.yaml`:

```yaml
hardware:
  integration_mode: tbank_aqsi   # tbank_aqsi | tbank_pos_sbp | legacy_umka

fiscal:
  enabled: false   # true только для legacy_umka

hardware:
  printer:
    enabled: false
```

| `integration_mode` | Поведение |
|--------------------|-----------|
| `tbank_aqsi` | Оплата и чек на aQsi, без УМКА/принтера |
| `tbank_pos_printer` | **POS + HS-K33 + облачный чек**, без УМКА |
| `tbank_pos_sbp` | То же + СБП QR на экране киоска |
| `legacy_umka` | УМКА HTTP + опционально HS-K33 |

## Что запросить в Т-Бизнесе (один звонок)

1. Для киоска без кассира: **aQsi 6** + полная **API-документация** для unattended (api@aqsi.ru).
2. Если остаётся PAX: модель, **Smart Sale**, IP, SDK для Windows.
3. Для СБП на большом экране: **TerminalKey** интернет-эквайринга + связка с фискализацией (aQsi или облачная касса).
4. Помощь с регистрацией ФН и ОФД на выбранном устройстве.

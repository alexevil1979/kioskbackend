# УМКА-01-ФА — онлайн-касса (опционально)

> **Статус:** не используется при `hardware.integration_mode: tbank_aqsi`.  
> Включайте только режим `legacy_umka`, если касса уже стоит на объекте.  
> См. [00-architecture-tbank-first.md](00-architecture-tbank-first.md).

## Назначение в киоске

Автоматизированная ККТ без кассира: фискальный чек после оплаты, передача ФД в ОФД.

## Подключение (ваша схема)

**Ethernet → switch** (та же LAN, что и NUC).

Комплектации (справочно):

| Версия | Связь | Примечание |
|--------|-------|------------|
| **Lite** | USB к host, печать на внешнем принтере | Host может печатать на совместимых принтерах |
| **Standard** | Ethernet, Wi‑Fi, Bluetooth | Предпочтительно для киоска по LAN |

Источники: [Астрал ККТ](https://astral.ru/shop/catalog/onlayn-kassa/pos-kompyuter-umka-01-fa-lite/), [kasscenter.ru](https://kasscenter.ru/product/umka-01-fa).

## Протокол интеграции (для разработки)

Платформа **УМКА** (arMax):

- Транспорт: **HTTP** (локально `http://<IP_кассы>:<порт>/...`)
- Авторизация: **HTTP Basic** (логин = номер кассира, пароль = пароль кассира; по умолчанию часто 1–99)
- Формат: **XML или JSON**
- Тестовый стенд: `http://office.armax.ru:58088` (общая тест-касса, не ваша!)

### Основные методы

| Метод | Назначение |
|-------|------------|
| `GET /cashboxstatus.json` | Состояние ККТ, смена, ФН, бумага |
| `GET /cycleopen.json?print=1` | Открытие смены |
| `POST /fiscalcheck.json` | **Пробитие чека** (основной для киоска) |

Полное описание: [umki.org — API УМКА](https://umki.org/knowledge-base/api-%d0%b4%d0%bb%d1%8f-%d1%80%d0%b0%d0%b1%d0%be%d1%82%d1%8b-%d1%81-%d0%ba%d0%ba%d0%bc-%d0%bd%d0%b0-%d0%bf%d0%bb%d0%b0%d1%82%d1%84%d0%be%d1%80%d0%bc%d0%b5-%d1%83%d0%bc%d0%ba%d0%b0/)

Краткая выжимка: [UMKA_API_QUICKREF.md](UMKA_API_QUICKREF.md)

## Реализация в проекте

- `src/services/fiscal_umka.py` — HTTP-клиент, проверка статуса, заготовка чека
- `config/settings.yaml` → `fiscal.*`, `hardware.umka.*`

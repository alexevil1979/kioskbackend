# aQsi 6 — основное устройство Т-Банка для киоска

## Зачем

Одно устройство вместо связки **POS + УМКА + принтер**:

- Торговый эквайринг Т-Банка
- Онлайн-касса 54-ФЗ
- Термопринтер чеков
- QR / карта / (СБП на устройстве — уточнить тариф)

[Страница Т-Банка](https://www.tbank.ru/business/payments/acquiring/terminal/) · [aQsi 6](https://aqsi.ru/aqsi6/)

## Подключение

- Wi‑Fi и/или мобильная связь (сим от Т-Банка)
- Для киоска: в одной сети с NUC (тот же роутер)

## Интеграция с PyQt-киоском

### Cloud API (рекомендуется)

1. Личный кабинет aQsi → Настройки → Интеграции → **Внешний API** → API-ключ  
   Инструкция: https://aqsi.ru/support/integraciya-po-vneshnemu-api/

2. Создание заказа с позициями (отложенный заказ синхронизируется на терминал):  
   https://api.aqsi.ru/#tag/Orders

3. Статус / чеки: https://api.aqsi.ru/#tag/Receipts

4. Поддержка разработчиков: **api@aqsi.ru**

### Unattended (важно для киоска!)

Стандартный сценарий aQsi — кассир выбирает «Отложенные заказы».  
Для **самообслуживания** нужно у Т-Банка/aQsi подтвердить:

- автоприём заказа без кассира;
- или оплату по API без ручного шага на терминале.

## СБП: экран киоска vs aQsi

| Подход | Плюс | Минус |
|--------|------|-------|
| QR на 32″ (Т‑Касса) | Крупный QR | Нужна отдельная фискализация после оплаты |
| QR на aQsi | Чек сразу на устройстве | Мелкий экран у терминала |

Для «только Т-Банк» предпочтительно **оплата и чек на aQsi**, киоск только инициирует заказ.

# Настройка сети: NUC (киоск) + принтер HS-K33 через switch (режим 2)

**Версия:** 1.0  
**Дата:** 2026-05-29  
**Назначение:** удалённая первичная настройка по **AnyDesk** при «чистой» Windows 10  
**Режим ПО:** `tbank_pos_printer` (резервный; основной — aQsi, режим 1)  

---

## 1. Исходное состояние (как на объекте)

| Устройство | Состояние |
|------------|-----------|
| **NUC** | Windows 10 Pro, чистая установка, ПО киоска ещё не установлено |
| **Связь NUC** | Только **Wi‑Fi** (для удалённого доступа **AnyDesk**) |
| **Ethernet NUC** | Кабель в **switch**, IP пока не настроен |
| **Принтер HS-K33** | Включён, кабель в **switch** |
| **Интернет-роутер** | Wi‑Fi для NUC; switch **не обязан** быть подключён к роутеру для LAN-связи NUC↔принтер |

```
        [ Роутер Wi‑Fi ]──── Wi‑Fi ────► NUC (AnyDesk, интернет)
              │
              │ (опционально, для CRM/облака позже)
              │
        [ Switch ]──── Ethernet ────► NUC
              │
              └──── Ethernet ────► HS-K33
```

> **Важно:** связь киоск ↔ принтер идёт **внутри switch** (L2). Роутер для печати не обязателен. Wi‑Fi нужен для AnyDesk и будущего API CRM.

---

## 2. План IP-адресов (рекомендуемый)

Используйте одну подсеть, **без DHCP на switch** (статика вручную).

| Устройство | IP | Маска | Шлюз | Примечание |
|------------|-----|-------|------|------------|
| NUC (Ethernet) | `192.168.10.10` | `255.255.255.0` | *(пусто)* | Только LAN к принтеру |
| HS-K33 | `192.168.10.101` | — | — | Задать на принтере или через утилиту |
| Шлюз | — | — | — | На LAN-адаптере **не указывать**, если нет выхода в другую сеть через switch |

*Если на объекте уже другая подсеть (например `192.168.1.x`) — используйте её единообразно в `config/settings.yaml`.*

---

## 3. Подготовка (на объекте / удалённо)

### 3.1. Физика

1. Принтер HS-K33 и NUC подключены к **одному switch** патч-кордами.
2. На принтере горит питание, рулон бумаги установлен (**80 мм**).
3. NUC подключён к **Wi‑Fi** роутера (интернет есть).
4. Установлен **AnyDesk** (или аналог), известен ID и пароль для доступа.

### 3.2. Узнать заводской IP принтера (если неизвестен)

Варианты:

- Распечатать **тестовую страницу** (кнопка FEED при включении — см. инструкцию HS-K33).
- Уточнить в паспорте / у поставщика (часто `192.168.1.100` или DHCP).
- Временно подключить принтер к роутеру и найти в списке DHCP клиентов.

Далее в инструкции предполагается, что вы назначите принтеру **`192.168.10.101`**.

## 4. Настройка Windows 10 на NUC (Ethernet)

Выполнять **по AnyDesk** под учётной записью администратора.

### Шаг 1. Открыть параметры сети

1. `Параметры` → **Сеть и Интернет** → **Ethernet** (или «Настройка параметров адаптера»).
2. Адаптер: **Ethernet** / **Подключение по локальной сети**.

### Шаг 2. Статический IP (без шлюза на LAN)

1. **Назначение IP** → **Вручную**.
2. Включить **IPv4**.
3. Заполнить:

| Поле | Значение |
|------|----------|
| IP-адрес | `192.168.10.10` |
| Маска подсети | `255.255.255.0` |
| Шлюз | *(оставить пустым)* |
| DNS | *(можно пусто для LAN; для Wi‑Fi DNS остаётся у Wi‑Fi адаптера)* |

4. Сохранить.

### Шаг 3. Отключить конфликт метрик (если есть проблемы с маршрутами)

1. `ncpa.cpl` → Ethernet → **Свойства** → **IP версии 4** → **Дополнительно**.
2. Снять галочку **Автоматическое назначение метрики** → метрика **50** (выше, чем Wi‑Fi).
3. Wi‑Fi адаптер: метрика **10** (чтобы интернет шёл через Wi‑Fi).

### Шаг 4. Проверка связи

Открыть **PowerShell** или **cmd**:

```powershell
ping 192.168.10.101 -n 4
```

Ожидание: `Ответ от 192.168.10.101`.  
Если **таймаут** — см. раздел 8 «Устранение неполадок».

### Шаг 5. Брандмауэр Windows

Разрешить исходящие к принтеру (для RAW TCP 9100):

```powershell
New-NetFirewallRule -DisplayName "Kiosk Printer RAW" -Direction Outbound -Action Allow -Protocol TCP -RemotePort 9100
```

Или временно для теста: отключить брандмауэр частной сети *(только на время отладки)*.

## 5. Настройка IP на принтере HS-K33

> Точные шаги зависят от прошивки. Используйте **Printer Tools** / **SDK** с сайта [hsprinter.com](https://www.hsprinter.com/en/download.html).

Общая цель:

| Параметр | Значение |
|----------|----------|
| IP | `192.168.10.101` |
| Маска | `255.255.255.0` |
| Порт печати (RAW) | **9100** |
| DHCP | **Выкл** (статический IP) |

После смены IP — снова `ping` с NUC.

## 6. Тест печати с NUC

### Вариант A: PowerShell (без драйвера)

```powershell
$printerIp = "192.168.10.101"
$port = 9100
$text = "TEST`nKiosk Farm`n192.168.10.10 -> printer OK`n`n`n"
$bytes = [System.Text.Encoding]::GetEncoding(866).GetBytes($text)
$client = New-Object System.Net.Sockets.TcpClient($printerIp, $port)
$stream = $client.GetStream()
$stream.Write($bytes, 0, $bytes.Length)
$stream.Flush()
$client.Close()
Write-Host "Sent to printer"
```

Должна выйти тестовая лента.

### Вариант B: Установка драйвера (опционально)

1. Скачать **Thermal Printer Driver** для Windows с hsprinter.com.
2. Создать порт **Standard TCP/IP** → `192.168.10.101`, порт `9100`.
3. Печать пробной страницы из Windows.

Для киоска достаточно **RAW TCP 9100** (как в `printer_hs_k33.py`).

## 7. Конфиг приложения киоска

После установки ПО киоска отредактировать `config/settings.yaml`:

```yaml
hardware:
  integration_mode: tbank_pos_printer
  printer:
    enabled: true
    host: "192.168.10.101"
    port: 9100
    connection: ethernet
  tbank_terminal:
    host: "192.168.1.102"   # IP POS — когда будет настроен
    port: 27015
    smart_sale_enabled: true
    use_mock: true            # false после SDK Т-Банка

fiscal:
  enabled: true
  provider: cloudkassir
  use_mock: true
```

Перезапустить киоск, провести тестовую «оплату» (mock) — должна печататься квитанция.

## 8. Устранение неполадок

| Симптом | Действие |
|---------|----------|
| `ping` не идёт | Проверить кабели, switch (индикаторы LINK), один VLAN, нет ли изоляции портов |
| `ping` идёт, печать нет | Проверить порт **9100**, брандмауэр, кодировка; утилита Printer Tools |
| Пропадает интернет | Не указывать шлюз на Ethernet; метрика Wi‑Fi ниже |
| AnyDesk отвалился | Wi‑Fi роутера; не отключать Wi‑Fi при настройке LAN |
| Два IP на одном кабеле | Только один активный Ethernet-адаптер в switch |
